# Implementation Planning

Before writing any code:

1. Read all authoritative documents.
2. Understand the requested milestone.
3. Break the work into phases.
4. List every file that will be created.
5. List every dependency and justify it.
6. Identify risks and assumptions.
7. Identify anything intentionally deferred.
8. Map the implementation to the milestone's Definition of Done.
9. Wait for approval before generating code.

Do not generate code during this planning phase.
